# INF-3 — Канонический хэш и воспроизводимость

- **Эпик:** E0 — Scaffolding & инфраструктура
- **Статус:** done
- **Зависимости:** INF-1
- **Реализует улучшение:** №4 (замена нестабильного `hash()` на sha256)
- **Критический путь:** да

## Описание

Стабильный детерминированный хэш для дедупликации и воспроизводимости.

- `stable_hash(obj) -> str` на `hashlib.sha256` от **canonical JSON**
  (sorted keys, нормализация float, стабильная сериализация datetime).
- Хелпер получения текущего git commit hash (для записи в результаты run).
- Утилиты сериализации доменных моделей в canonical-форму.

## Контекст

Встроенный `hash()` для строк рандомизирован между процессами
(`PYTHONHASHSEED`) — он непригоден для `run_hash`/cache/resume.

## Критерии приёмки (DoD)

- [ ] Один и тот же объект даёт одинаковый хэш между процессами
      (тест запускает подпроцесс и сравнивает).
- [ ] Порядок ключей в dict не влияет на результат.
- [ ] Незначащие различия float-представления нормализуются.
- [ ] git commit hash доступен (или явный fallback вне git).
