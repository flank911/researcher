# BT-5 — Risk: маржа/ликвидация, intrabar TP/SL

- **Эпик:** E2 — Backtest Core
- **Статус:** done
- **Зависимости:** BT-4
- **Реализует улучшение:** №5 (intrabar), №10 (ликвидация/маржа)
- **Критический путь:** да

## Описание

Риск-слой движка для leverage и внутрибарного исполнения.

- `backtest/risk` (RiskManager): модель маржи и ликвидации позиции.
- Внутрибарное срабатывание TP/SL и trailing-стопов.
- Зафиксированное **пессимистичное** допущение: если на одном баре задеты и TP,
  и SL — срабатывает SL первым (документировать).

## Критерии приёмки (DoD)

- [x] Позиция ликвидируется при пробое маржи; equity не уходит в нереалистичный минус.
- [x] Тест одновременного задевания TP и SL → срабатывает SL (пессимистично).
- [x] Trailing-стоп корректно отслеживает экстремум пути.
- [x] Документировано intrabar-допущение.

## Примечание

- `backtest/risk.py` (`RiskManager`): кросс-маржинальная ликвидация + внутрибарные
  TP/SL/trailing по `high`/`low`. Пессимистичные допущения (порядок движения
  хуже→лучше ⇒ SL раньше TP; trailing от экстремума к началу бара; гэп сквозь
  уровень ⇒ исполнение по `open`) задокументированы в докстринге модуля.
- Ликвидация активна при наличии `high`/`low`; при заданных TP/SL/trailing их
  отсутствие — ошибка. Выходы риск-слоя проходят через ту же модель комиссий и
  слиппеджа, что и сигнальные (рыночные, taker).
- `risk_based` sizing (отложен из BT-4) реализован: объём подбирается под
  `position_size_value`% риска на дистанции `stop_loss_pct`, экспозиция ограничена
  плечом; без `stop_loss_pct` — ошибка.
