# BT-3 — Fill timing / анти-look-ahead

- **Эпик:** E2 — Backtest Core
- **Статус:** done
- **Зависимости:** BT-2
- **Реализует улучшение:** №3 (исполнение на следующем баре)
- **Критический путь:** да

## Описание

Корректная модель времени исполнения, исключающая заглядывание в будущее.

- Сигнал, сформированный на `close` бара N, исполняется на `open` бара N+1.
- Параметры `signal_lag` / `fill_on` в `ExecutionModel`.
- Гарантия: ни одно решение не использует данные, недоступные на момент решения.

## Критерии приёмки (DoD)

- [ ] Тест доказывает: сигнал на `close` НЕ исполняется на том же баре.
- [ ] Изменение `signal_lag` сдвигает точку исполнения предсказуемо.
- [ ] Регрессионный тест на отсутствие look-ahead в индикаторах + исполнении.
